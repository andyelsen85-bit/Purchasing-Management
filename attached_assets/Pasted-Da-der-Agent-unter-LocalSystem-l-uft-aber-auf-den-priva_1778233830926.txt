Da der Agent unter LocalSystem läuft, aber auf den privaten Schlüssel im Store des Benutzers zugreifen muss, ist der Einsatz von PsExec die effizienteste Lösung. LocalSystem kann den Schlüssel des Nutzers aufgrund der DPAPI-Verschlüsselung nicht direkt lesen; der Befehl muss daher zwingend in der aktiven Sitzung des Nutzers ausgeführt werden.Umsetzung mit PsExecMit PsExec können Sie einen Prozess in einer spezifischen Session-ID starten. Da der Benutzer bereits eingeloggt ist, nutzen Sie dessen Session (meist Session 1 auf Single-User-Systemen).Session-ID ermitteln: Der Agent muss zuerst herausfinden, welche Session-ID der aktive Nutzer hat.powershell$sessionID = (quser | Select-String ">" | ForEach-Object { $_ -split '\s+' } | Select-Object -Index 2)
Verwende Code mit Vorsicht.Befehl in der User-Session ausführen: Nutzen Sie den Parameter -i (interaktiv) gefolgt von der Session-ID, um die Signatur unter dem User-Kontext auszuführen.powershell# Beispiel: Signieren eines Skripts mit dem User-Zertifikat
psexec.exe -i $sessionID powershell.exe -Command "Set-AuthenticodeSignature -FilePath 'C:\Pfad\Dokument.ps1' -Certificate (Get-ChildItem Cert:\CurrentUser\My -CodeSigningCert)[0]"
Verwende Code mit Vorsicht.Alternative: Geplanter Task (Native Lösung)Falls Sie kein externes Tool wie PsExec installieren möchten, kann der Agent einen Scheduled Task erstellen, der sofort unter dem Account des angemeldeten Benutzers ausgeführt wird.Vorteil: Nutzt integrierte Windows-Mechanismen.Einstellung: Der Task muss so konfiguriert sein, dass er "Nur ausgeführt wird, wenn der Benutzer angemeldet ist". Dadurch erhält der Prozess automatisch Zugriff auf den Cert:\CurrentUser-Store und die privaten Schlüssel des Nutzers.SicherheitshinweisStellen Sie sicher, dass das Zertifikat im User-Store tatsächlich den privaten Schlüssel enthält und für "Code Signing" oder "Document Signing" vorgesehen ist. Ohne den privaten Schlüssel schlägt jede Signaturoperation mit einer CryptographicException fehl.Soll die Lösung vollautomatisch im Hintergrund ohne jegliche Nutzerinteraktion (Pop-ups) laufen, oder darf der Nutzer kurz ein Fenster sehen?Weitere hilfreiche Quellen:PsExec Dokumentation bei Microsoft LearnAnleitung zum Signieren von Skripten (Microsoft Learn)
